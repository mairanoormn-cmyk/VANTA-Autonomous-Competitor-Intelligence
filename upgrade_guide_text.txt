2.1  What It Does
MCP Server is Bright Data's Model Context Protocol server. It lets your AI agent (Gemini) directly connect to the live web through a standardized interface — no custom scraping code needed per-site. Instead of calling SERP API and Web Unlocker separately with raw HTTP requests, your agent uses MCP tools natively in its tool-calling loop. This means it can access LinkedIn, Reddit, Glassdoor, G2, and any public website in one unified call.
2.2  Why It Is Critical
The hackathon judging criteria specifically lists 'demonstrable Bright Data product use'. MCP Server is listed first on every track page.
Winning projects (CompeteIQ, Agent Atlas, Cortex) all used MCP Server. Projects without it scored lower.
It replaces your current manual tool_call loop with a cleaner, more powerful architecture — agents become more autonomous and impressive to judges.
Allows your agent to access sites it currently cannot (LinkedIn, Glassdoor) without writing separate scrapers.

Criticality

Prize Impact
Adds a 3rd Bright Data tool to your submission. Judges count tools used — more tools = higher Application of Technology score.

3.1  What It Does
Scraping Browser is a full cloud-based Chromium browser controlled via Puppeteer/Playwright API. Unlike Web Unlocker (which fetches raw HTML), Scraping Browser fully renders JavaScript before returning the page. This is essential for sites like G2.com, Glassdoor, Trustpilot, and LinkedIn — all of which load their review content dynamically after initial page load.
3.2  Why It Is Critical
G2.com and Glassdoor are the single most valuable sources for competitor dissatisfaction signals. They are 100% JS-rendered and will return empty content with Web Unlocker alone.
Adding Scraping Browser increases your Bright Data tool count from 2 to 3, which directly impacts the 'Application of Technology' judging score.
The signal quality (and therefore your intent scores) will be dramatically more accurate with real review data.
3.3  How to Implement It
Add a new function scrape_js_site() in backend/agent.py alongside your existing scrape_url():
async def scrape_js_site(url: str, wait_selector: str = None) -> str:
    """
    Use Bright Data Scraping Browser for JS-rendered pages.
    Best for: G2.com, Glassdoor, Trustpilot, LinkedIn, Reddit new UI.
    """
    payload = {
        "zone": os.getenv("BRIGHT_DATA_SB_ZONE", "scraping_browser1"),
        "url": url,
        "format": "raw",
        "render_js": True,
    }
    if wait_selector:
        payload["wait_for_selector"] = wait_selector
    async with httpx.AsyncClient(timeout=60) as client:
        response = await client.post(
            "https://api.brightdata.com/request",
            headers={"Authorization": f"Bearer {BRIGHT_DATA_API_KEY}"},
            json=payload
        )
    return response.text

7.3  How to Implement It
In agent_orchestrator.py, add a signal fusion function:

def fuse_signals(signals: list) -> list:
    """
    Fuse duplicate company signals from multiple sources.
    Rule: If same company+pain_point appears in 2+ sources, increase intent_score by 2.
    """
    signal_map = {}  # Key: (company_name, pain_point)
    
    for sig in signals:
        key = (sig['company_name'].lower(), sig['pain_point'].lower())
        if key not in signal_map:
            signal_map[key] = sig.copy()
            signal_map[key]['source_count'] = 1
            signal_map[key]['sources'] = [sig['source']]
        else:
            signal_map[key]['source_count'] += 1
            signal_map[key]['sources'].append(sig['source'])
            # Boost score for multi-source signals
            signal_map[key]['intent_score'] = min(10, signal_map[key]['intent_score'] + 2)
    
    fused = list(signal_map.values())
    return sorted(fused, key=lambda s: s['intent_score'], reverse=True)

Then call it before yielding signals_ready:

# In run_agent_stream(), before: yield {"type": "signals_ready", "signals": signals}

yield {"type": "signals_ready", "signals": signals}

async def recall_similar(company: str, competitor: str) -> str:
    """Query memory for patterns related to this company or competitor."""
    results = await cognee.search(
        f'companies leaving {competitor} with similar pain points to {company}',
        query_type='INSIGHTS'
    )
    if not results:
        return 'No historical patterns found.'
    return ' | '.join([r.get('text', '') for r in results[:3]])

Criticality
🟠 HIGH — Strongest intent signal available. VEINIntel (a top submission) built their entire pitch around this.
Prize Impact

5.1  What It Does
This feature searches LinkedIn job postings for companies that are actively hiring for roles that indicate they are switching away from your tracked competitor. For example, if a company is hiring a 'Salesforce Alternative Implementation Specialist' or a 'HubSpot Migration Lead', that is a near-certain signal they are leaving Salesforce. This is the highest-quality intent signal available because it requires budget approval — the company has already committed to switching.
5.2  The Logic — How Hiring = Churn Signal
Company posts 'CRM Implementation' role + mentions competitor by name in 'not required' = actively migrating
Company posts multiple RevOps roles suddenly = GTM tooling overhaul in progress
Company posts 'Salesforce Admin' role with 'Salesforce optional' in description = they want to reduce Salesforce dependency
5.3  How to Implement It
Add a new function in agent.py using Bright Data's LinkedIn Jobs pre-built scraper (Web Scraper API):
async def detect_hiring_signals(competitor: str) -> list[dict]:
    """
    Use Bright Data Web Scraper API to pull LinkedIn job postings
    that indicate a company is migrating away from a competitor.
    """
    # Trigger the LinkedIn Jobs dataset
    trigger_response = await httpx.post(
        "https://api.brightdata.com/datasets/v3/trigger",
        headers={
            "Authorization": f"Bearer {BRIGHT_DATA_API_KEY}",
            "Content-Type": "application/json"
        },
        json={
            "dataset_id": "gd_l1viktl72bvl7bjuj0",
            'filters': {
                "keyword": f"CRM implementation NOT {competitor}",
                "location": "United States",
                "date_posted": "past_month"
            }
        }
    )
    snapshot_id = trigger_response.json()['snapshot_id']
    # Poll for results (retry up to 5 times with 10s delay)
    for _ in range(5):
        await asyncio.sleep(10)
        result = await httpx.get(
            f"https://api.brightdata.com/datasets/v3/snapshot/{snapshot_id}",
            headers={'Authorization': f'Bearer {BRIGHT_DATA_API_KEY}'}
        )
        if result.json().get('status') == 'ready':
            return parse_hiring_signals(result.json()['data'], competitor)
    return []

Add a parser that extracts company name, job title, and infers the migration intent score:
def parse_hiring_signals(jobs: list, competitor: str) -> list[dict]:
    signals = []
    migration_keywords = [
        'implementation', 'migration', 'transition', 'alternative',
        'replace', 'modernize', 'consolidate'
    ]
    for job in jobs:
        description = job.get('description', '').lower()
        # Boost score if competitor name appears in 'not required'
        comp_in_desc = competitor.lower() in description
        has_migration_kw = any(kw in description for kw in migration_keywords)
        if has_migration_kw:
            signals.append({
                'company_name': job.get('company_name'),
                'job_title': job.get('title'),
                'source': 'LinkedIn Jobs',
                'pain_point': 'Active CRM migration hiring',
                'intent_score': 9 if comp_in_desc else 7
            })
    return signals

In backend, update the /api/competitor-stats endpoint to return the 5 dimension scores by categorizing signal pain_point values:
def compute_dimension_scores(signals: list) -> dict:
    categories = {
        'pricing_score': ['pricing', 'cost', 'expensive', 'license'],
        'support_score': ['support', 'service', 'response', 'help'],
        'feature_score': ['feature', 'missing', 'functionality', 'limitation'],
        'hiring_score':  ['hiring', 'migration', 'linkedin', 'job posting'],
        'sentiment_score': ['reddit', 'twitter', 'glassdoor', 'g2'],
    }
    scores = {k: 0 for k in categories}
    for signal in signals:
        pain = signal.pain_point.lower()
        for dimension, keywords in categories.items():
            if any(kw in pain for kw in keywords):
                scores[dimension] += signal.intent_score * 2
    # Normalize to 0-100
    return {k: min(v, 100) for k, v in scores.items()}

const ROIPanel = ({ signals }) => {
  const highIntent = signals.filter(s => s.intent_score >= 7);
  const potentialRevenue = highIntent.length * 45000 * 0.23;
  const hoursSaved     = signals.length * 8;

    ? Math.round((potentialRevenue / costSaved) * 100) : 0;

  return (
